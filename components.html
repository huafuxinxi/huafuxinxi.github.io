# 搜索组件

# 简介

搜索组件用于整个项目的搜索，包括：文本搜索，下拉选择搜索，时间搜索，时间范围搜索，日期搜索，日期范围搜索等。该组件已经全局注册，直接使用即可。使用方式如下示例所示：

<search-list :inputList="inputList" @on-search="search"></search-list>
1

# inputList属性

inputList是一个对象数组，数组的length表示有几项搜索框，数组中的每一项都是一个对象，对象包含以下属性：

属性名 类型 是否必填 描述
name string 是 展示输入框的类型，暂时只有一下TIP中的八种选项，其它选项不识别
bindKey string 是 搜索后返回的key值
placeholder string 否 搜索框为空时占用文本
value string 是 用于v-model的绑定，初始值
title string 否 搜索标题
titleWidth number 否 标题宽度（默认自适应）
inputWidth number 否 输入框宽度（默认200px）
hidden boolean 否 该搜索框是否显示

TIP

name属性暂时有8种选项：

  • text-input：文本输入框
  • select-input：下拉选择框
  • time-input：时间选择器(时分秒)
  • time-range-input：时间范围选择器(时分秒)
  • date-input：日期选择器(年月日)
  • date-range-input：日期范围选择器(年月日)
  • date-time-input：日期时间选择器(年月日时分秒)
  • date-time-range-input：日期时间范围选择器(年月日时分秒)

# name属性

name属性提供了8种选项，根据选项的不同，需要提供不同的额外字段：

name类型 需添加或修改字段 描述
select-input selectList 下拉数据，数据结构同el-select组件
time-range-input bindKey 搜索返回绑定的key值数组，例如：['startTime', 'endTime']
time-range-input value 搜索给定的初始值，要么是'',表示没有初始值，要么是['00:00:00', '12:16:19']这种，不可以是空数组和['', '']
date-range-input bindKey 搜索返回绑定的key值数组，例如：['startDate', 'endDate']
date-range-input value 搜索给定的初始值，要么是'',表示没有初始值，要么是['2019-02-12', '2020-01-06']这种，不可以是空数组和['', '']
date-time-range-input bindKey 搜索返回绑定的key值数组，例如：['startDateTime', 'endDateTime']
date-time-range-input value 搜索给定的初始值，要么是'',表示没有初始值，要么是['2019-02-12 00:00:00', '2020-01-06 12:16:19']这种，不可以是空数组和['', '']

# @on-search属性

当使用搜索组件时，会通过@on-search的方式向父组件传递bindKey绑定的搜索key值。只需要在父组件中使用方法接受即可。

# @on-reset属性

当使用搜索组件时，会通过@on-search的方式向父组件重置（清空）搜索选项，传递给父组件的参数为空对象{}。

# 示例

<template>
    <div>
        <search-list :inputList="inputList" @on-search="search"></search-list>
    </div>
</template>
<script lang='ts'>
import { Component, Vue } from 'vue-property-decorator'

@Component
export default class Test extends Vue {
    private inputList = [
        {
        name: 'text-input',
        bindKey: 'personCode',
        placeholder: '请输入人员编号',
        value: '',
        title: '人员编号：'
        },
        {
            name: 'text-input',
            bindKey: 'personName',
            placeholder: '请输入人员姓名',
            value: '',
            title: '人员姓名：'
        }
    ]
    private search(data: object) {
        console.log(data)
    }
}
</script>

# 表格组件

# 简介

表格组件是封装了el-table和el-pagination，表格组件由标题区、列表区、分页区三大部分组成。使用方式如下示例所示：

<hf-table
    :columns="tableColums"
    :data="logList"
    :tableTitle="tableTitle"
    :page="page"
    :load="load"
    :pageSize="pageSize"
    :total="count"
    @current-change="currentChange"
    @size-change="sizeChange"
></hf-table>

# 属性概览

属性（方法）名 类型 是否必填 描述
columns array{object} 是 列表头
data array{object} 是 列表数据，与el-table相同
tableTitle string 否 列表标题
page number 否 当前页，不填则默认为第一页
load boolean 否 loading效果
pageSize number 否 每页个数，不填则默认10个
total number 是 表格数据总数，用于分页
options object 否 表格配置属性，包含斑马纹，高亮，选择框等
current-change function 否 切换页触发，返回切换第几页
size-change function 否 改变每页个数触发，返回切换多少个
selection-change function 否 切换选择框时触发，返回当前选择数据

# columns

列表头的描述项，是一个数组，数组的length代表有多少列，数组中的每项都是一个对象，其中包含属性：

属性名 类型 是否必填 描述
title string 是 列名
key string 是 列的key值，对应data中的属性
width number 否 列宽
minWidth number 否 最小列宽
align left/right/center 否 对齐方式，默认居中
formatter function 否 自定义格式化列表展示，使用jsx语法，第一个参数为每行的数据
render function 否 自定义render组件，参照render组件写法

TIP

formatter和render如果不熟悉，请参考下面的示例

export const getTableColums = (that: any) => [
    {
        title: '头像',
        key: 'personImage',
        width: 96,
        align: 'center',
        formatter: (row: any) => 
        `<img src="
        ${row.personImage || null}
        " class="person-image" onerror="javascript:this.src='${that.errImg}'"/>`
    },
    {
        title: '人员编号',
        key: 'personCode',
        minWidth: 120,
    },
    {
        title: '操作',
        minWidth: 250,
        render: (h: any, params: any) => {

            return h('div', {
                attrs: {
                    class: 'operate-content'
                }
            }, [
                h('span', {
                    attrs: {
                        class: 'edit-operate'
                    },
                    on: {
                        click: () => { that.editPerson(params.row) }
                    }
                }, '编辑'),
                h('span', {
                    attrs: {
                        class: 'remove-operate'
                    },
                    on: {
                        click: () => { that.removePerson(params.row) }
                    }
                }, '删除'),
                h('span', {
                    attrs: {
                        class: 'recognition-operate'
                    },
                    on: {
                        click: () => { that.goRecognition(params.row) }
                    }
                }, '识别记录')
            ])
        }
    }
]

# options

options是整个表格的配置项，其中包含以下四种属性（均为boolean）：

  • stripe： 是否有斑马纹
  • border： 是否带有纵向边框
  • highlightCurrentRow： 是否高亮当前行
  • selection： 是否有选择框
